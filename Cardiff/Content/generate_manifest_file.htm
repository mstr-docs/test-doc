Generate the Manifest File Using Workstation

Before deploying the HyperIntelligence for Office add-in, you must generate a manifest file from an environment in Workstation. Manifest files are directly tied to the environments they come from, so make sure that your environment contains all of the cards you want to see in Outlook.

To generate a manifest file, you must have a MicroStrategy environment in Workstation. For steps, see Introduction to Environments in Workstation Online Help.

Your environment must be an HTTPS domain. Outlook does not support add-ins connecting to HTTP domains.

How to Generate the Add-in Manifest File Using Workstation

  • Windows
  • Mac

  1. In Workstation, click the Environments tab.

  2. Right-click the environment you want to use and select Properties.
  3. Click Generate Add-in File next to MicroStrategy HyperIntelligence.

  4. From the Generate Add-in File window you have two options:

    1. Edit the name of the add-in.

      Click the pencil icon to edit the name of the add-in that will appear in your Outlook ribbon.

      You do not have to change the name of the add-in to download the manifest file.

    2. Enter a URL to authorize third-party authentication service domains for the add-in to function properly. Click Add a URL if you have multiple authentication service domains.

      This option only applies to environments that use SAML or Trusted authentication. For steps to enable these authentication methods, see Enabling Single Sign-On Authentication.

  5. Click Download.
  6. Save the manifest file on your hard drive.

  1. In Workstation, click the Environments tab.

  2. Right-click the environment you want to use and select Get info.
  3. Expand the HyperIntelligence Add-in drop-down and select Generate Add-in File.
  4. From the Generate Add-in File window you have two options:
    1. Edit the name of the add-in.

      You do not have to change the name of the add-in to download the manifest file.

    2. Enter a URL to authorize third-party authentication service domains for the add-in to function properly. Click Add a URL if you have multiple authentication service domains.

      This option only applies to environments that use SAML or Trusted authentication.

  5. Click Download.
  6. Save the manifest file on your hard drive.

Once you've generated the manifest file, you can deploy the HyperIntelligence for Office add-in as an administrator to multiple user groups, or deploy it directly to the Outlook desktop app. For instructions, see How to Deploy the HyperIntelligence for Office Add-in.
